Crosby

A Distributed Command Line Caching Layer

What is Crosby?

Crosby is a distributed command line caching layer. Are you tired of lengthy compilations or installations? Crosby checks if another Crosby user has done what you're attempting to do before and will write the results to your filesystem without actually running the command.

Try Now

Crosby supports Mac, Windows, Linux, FreeBSD, & OpenBSD in a variety of cpu architecutres.

Download Page →

How does it work?

Crosby looks for the result of the same command run against the same target. A target is defined as an identical file structure with identical contents in the current working directory that also has a matching operating system, CPU architecture, & operating system.

If a matching result is not found, Crosby runs the command normally and stores the result so that it will be available next time.

Usage

$ crosby <command> [args...]

Examples

$ crosby make
$ crosby gm mogrify -resize 120x120 *.jpg
$ crosby npm install
$ crosby gcc app.c
